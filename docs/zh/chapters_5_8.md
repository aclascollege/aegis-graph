# 第5章：主权与隐私保护

Aegis-Graph 的基石是**学术主权**。本章探讨确保全球教育“真相”不被单一实体控制的技术框架。

## 🔒 ZKE 隐私模型
在主权架构中，隐私是核心。Aegis-Graph 实现了 **零知识证据 (ZKE)** 模型：
*   **审计不留痕**：系统仅核验证据的“真实性”，而不存储或传输任何学生个人隐私数据 (PII)。
*   **加密可追溯**：核验路径被加密哈希处理，在不暴露原始数据的情况下允许未来追溯。

# 第6章：共识机制详解

Aegis-Graph 的最终审计结论是 **MARS 集群共识握手** 的结果。

## ⚖️ 加权投票模型
每个智能体（视觉、图谱、逻辑）根据其证据强度贡献投票权重：
*   **视觉 (VF)**：权重 0.30
*   **图谱 (GN)**：权重 0.35
*   **逻辑 (LA)**：权重 0.35
只有当共识分数 `T > 0.90` 时，系统才会签发主权核验凭证。

# 第7章：合规性与伦理框架

## 🛡️ 全球隐私合规
Aegis-Graph 的设计超越了 **GDPR (欧盟)**、**CCPA (美国)** 和 **PIPL (中国)** 的合规要求。我们通过 ZKE 技术确保核验过程符合最高标准的个人数据保护规范。

# 第8章：系统部署指南

## 🚀 启动主权节点
1.  **硬件要求**：建议使用 8核 CPU, 16GB RAM 及支持 CUDA 的 GPU。
2.  **Docker 部署**：
    ```bash
    docker pull aegis/kernel:latest
    docker run -d --name aegis-node -p 8080:8080 aegis/kernel
    ```
3.  **节点接入**：在配置文件中输入您的机构公钥，即可加入全球主权核验网络。
